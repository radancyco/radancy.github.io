YouTube embed not working in iOS Safari and Chrome (Fix)

Posted on Thursday, April 30, 2015 12:38:00 AM EDT by Michael "Spell" Spellacy

iPhone 5s Model: ME341LL/A iOS Version: 8.3 (12F0)

There appears to be a bug within iOS WebKit (and Blink) rendering engines. Under certain conditions, YouTube videos embedded within iframes cannot be viewed when a user clicks on the video. Eep!

In TMP's particular instance, the conditions had to be just right. A "perfect storm" of issues, if you will. :)

The Bug

Applying a universal selector to the :active pseudo-class with a property of -webkit-tap-highlight-color seemes to be the culprit.


*:active {
-webkit-tap-highlight-color: tomato;
}

Now, here is the odd part. The bug is only triggered when the above CSS block is present and there is an input element present on the page with a type attribute value of "search". I know, crazy, right?


<input type="search" placeholder="Search"/>

The Solution

Either renaming the input type to "text" or removing -webkit-tap-highlight-color (likely preferred solution) should alleviate the issue.
