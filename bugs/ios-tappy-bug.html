YouTube embed not working in iOS Safari and Chrome (Fix)

There appears to be a bug within iOS webkit related browsers where, under certain conditions, YouTube videos embedded within iframes cannot be viewed when a user clicks on the video. Eep!

In our particular instance, the conditions had to be just right. A "perfect storm" of bugs, if you will. :)

The Bug

Applying a universal selector to the :active pseudo-class with a declaration property of -webkit-tap-highlight-color seemed to be our culprit.


*:active {
-webkit-tap-highlight-color: tomato;
}

Now, here is the odd part. The bug is only triggered when the above CSS block is present and there is an input element present on the page with a type attribute value of "search". I know, crazy, right?


<input type="search" placeholder="Search"/>

The Solution

Either renaming the input type to "text" or removing -webkit-tap-highlight-color should alleviate the problem.
