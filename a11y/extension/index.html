Overview

Provide text alternatives for non-text content. Understanding 1.1.1

Basic Rules

  1. All images require an alt attribute, whether they are informative or decorative.
  2. If image is informative or contextually important to surrounding content, then it should have an alternative text value.
  3. If image is purely decorative, then it rquires no alternative text value.
  4. If image is contained within a hyperlink, then it must contain functional alternative text (e.g, My Awesome Vacation (Video))

Tip: Alternative text should be written in a descriptive, but succinct manner. Think of how you might describe the image to a friend who is blind.

Note: Unless it is relevant to the image, it is forbidden to intentionally include keywords in alternative text descriptions for the benefit of SEO. Websites are for people; not machines.

Overview

Provide captions for videos with audio Understanding 1.2.2

Test Procedure

Video on a page should be manually reviewed. If the video includes dialogue or important information, then it must contain captions or an audio description. Accuracy is also critical. If a video is found to have inaccurate captions, new captions must be produced.

Overview

Text can be resized to 200% without loss of content or function. Understanding 1.4.4

Test Procedure

Use Control or Command and Plus (+) keys to increase zoom level of viewport window. When 200% is reached, test to see if page content can still be accessed and is operabable. If zoom is disabled on a page, it is an immediated failure.

Provide user controls for moving content. Understanding 2.2.2

Test Procedure

Anthing that automatically moves on a web page, such as a carousel or video, must have a pause button. Web animation is exempt from this rule, but it is strongly reccomended that development leverage prefers-reduced-motion in their work.

Overview

Provide a 'Skip to Content' link. Understanding 2.4.1

Test Procedure

Areas of a web page that may be cumbersome to navigate through via keyboard or assistive technology, must provide a mechanisim to skip over that content. These "Skip" links should be available to all users when needed and not obscure other content on the page when visible. You will often find these links placed before a websites primary navigation.

Overview

Ensure keyboard focus is visible and clear. Understanding 2.4.7

Test Procedure

Tab through and engage with interactive components on the page to review their focus states. Default focus outlines are a browser feature, meant to aid those with low vision, and should never be removed without an adequate replacement. Outlines can be redesigned or an alternative focus state can be used. Any redesigned outline and all focus state must meet a 3:1 contrast ratio.

Related issues to be aware of when testing focus are the order in which elements are in when tabbed to. If tab stops are not in a fairly linear order, then this should be flagged. An interactive element with a tabindex other than 0 or -1 will interfere with a pages natural tab order.
